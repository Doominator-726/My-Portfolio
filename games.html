About

Video game development is what originally got be interested in computer science. At a first, it was a matter of design, but now I am pulled in by the technical elements: Making virtual worlds with various systems that connect. I love game development as an outlet for creativity and computer science. I have created various games across two engines, Godot and Pygame. Below you can find the various games that I have made.

Cultus Astrorum is a first person shooter demo I made for my IB CAS project. With the game, I got to learn a lot about game development. The game features pathfinding enemies with skeletal animations, multiple weapons, cutscenes, objectives, etc. The game also features a saving system which allows for saving the entire state of the game (it saves the enemy positions, present objects, everything). One thing I focused on was environmental interaction. In the game, you can break crates, pick up weapons and items, eat food, flush toilets, and toggle sinks. I made various 3D models for the game. For performence, baked lighting is also utilized. Overall, this was a very fun and challenging project that allowed me to improve at 3D game development.

This is a demo for a combat focused racing game. In the game, the player drives around a track, passing through each checkpoint and fighting other cars. The game has five weapons that can be used against your opponents including miniguns, rockets, and mines. You can also pick up items to repair your car from damage. As a car gets more damaged, its performence is reduced, and a fire on it begins to grow larger. There are also physics objects you can run into, including explosive barrels. The game has A.I. opponents who will attack the player and race around the track. The checkpoint system is used to track race placements for the player and end rankings. Overall, this game was my first deep dive into 3D game development and I created a demo that I am satisfied with.

Dwarfenstein is a "pseudo-3D" FPS rouge-like Pygame. In the game, the player goes through three sets of randomly generated dungeons before fighting a boss. The game has four weapons to find and ten enemies (including the final boss). The most notable thing about the game is the 3D effect. Created with help from a tutorial by Youtuber Coder Space the game uses raycasting and other techniques to give the appearence of a 3D space in a technically 2D environment. The game has floors, objects, items, and enemies. The other thing to note is the randomly generated dungeons. The game uses a random walk algorithm to generated maze levels for the player to navigate. For this feature I was inspired by a Unity tutorial but I did all of the coding alone. The game's enemies fight the player using pathfinding capabilities (also based on the Coder Space tutorial). I also feel like I learned a lot from doing the art for the game. I made use of color theory to create colorful and striking art. I feel that this is my best looking game.

ME is an exploration platformer game. In the game you go through a level with traps and enemies. In the level, you must collect four different upgrades as each is neccessary to progress in the level. The upgrades range from making certain platforms and enemies visible to giving the player the power to swap gravity. At the end you fight a boss based on myself. The most notable thing about this game is the unsettling nature of the game's graphics. The game features animated enemies, traps and projectiles. These entities use graphics I made using pictures of myself. The game looks weird yet the style is cohesive and unique. I made this game to familarize myself with the Godot engine. In this respect, the project was a success since I feel I improved my capabilities with the engine over the course of this game's development.

Minigolf Mayhem is a minigolf game demo with complex level design. It generates maps using .csv files created in a tile based map editor. You navigate these maps using a golf ball that you can swing around. However, you only have a limited number of swings and lives. You must also avoid obstacles such as holes, fire, and water. There are various contraptions in the maps such as arrows which control the direction of the ball, electric lamps that turn on and off, etc. The ball behaves differently on different surfaces. It receives the most friction on sand and the least on ice. The maps are also fitted with various decorations which fade when close to the ball.

Alien game is an action arcade game about staying alive via managing various elements. Your health is represented by a red bar, it is constantly depleting. Killing enemies will give you health but reduce your size. To regain size you must collect the power ups by shooting them. There are two types of projectiles. Teleports collect the power ups but will hurt you if used on enemies. Bullets kill the enemies but will destroy the power ups. In terms of theming, the game is presented as if it was made by aliens. It uses alien language with images and music we have beamed into space.

This was a game I made for my AP CSP Create Task. It was inspired by the 1971 game The Oregon Trail. The game mixes the original game with mad libs in a text based adventure. The game's structure is split up into days. Each day you can gather supplies before proceeding into a random event. These random events can include combat, trading, decisions, etc. They are preceded by randomly selected introductory events, such as stopping at a sign to choose a direction to go in. The game has a character system where each character has four stats that determine combat and their actions in certain situations. They also each have their own health values. You start out with five characters but you can gain more, or lose some to combat and disease. The game is filled with various locations, character names, and objects randomly chosen from large lists of strings. You may have Batman find various copies of 1984 one night or go into combat with Napoleon. Overall, this was a fun project and I learned a lot about Python and random generation.
